# Система Управления Задачами

## Описание проекта
Тестовое задание.
Система Управления Задачами (Task Management System) предназначена для управления задачами с возможностью создания, редактирования, удаления и просмотра. Проект реализует только API и поддерживает следующие возможности:

- Управление задачами (CRUD операции).
- Аутентификация и авторизация пользователей с использованием JWT токенов.
- Ролевая модель доступа (администратор и пользователь).
- Поддержка фильтрации и пагинации при выводе задач.
- Валидация входящих данных и обработка ошибок с предоставлением понятных сообщений.
- Документирование API с помощью Swagger UI и OpenAPI.
- Возможность локального запуска через Docker Compose.

Каждая задача включает следующие данные:
- Заголовок
- Описание
- Статус ("в ожидании", "в процессе", "завершено")
- Приоритет ("высокий", "средний", "низкий")
- Автор задачи и исполнитель
- Комментарии

### Основные возможности:
1. Аутентификация пользователей по username, email и паролю.
2. Администратор может:
    - Управлять всеми задачами.
    - Менять статус и приоритет задач.
    - Назначать исполнителей.
    - Оставлять комментарии.
3. Пользователь может:
    - Управлять задачами, где он назначен исполнителем.
    - Менять статус и оставлять комментарии.
4. Поддержка PostgreSQL для хранения данных.
5. Взаимодействие между сервисами через Spring Cloud Eureka и Gateway.
6. Базовые тесты для проверки функциональности системы.

## Используемые технологии
- **Java 17+**
- **Spring Boot**
- **Spring Security**
- **Keycloak** (OAuth2)
- **PostgreSQL**
- **Spring Cloud: Eureka, Gateway**
- **Docker Compose**
- **Swagger UI**

## Структура проекта
Проект состоит из следующих микросервисов:
- **Keycloak**: Сервер аутентификации.
- **Postgres User**: База данных для хранения данных пользователей.
- **Postgres Task**: База данных для хранения данных задач.
- **Eureka**: Сервер обнаружения микросервисов.
- **Account Service**: Сервис управления пользователями.
- **Task Service**: Сервис управления задачами.
- **Gateway**: API Gateway для маршрутизации запросов.

## Запуск проекта

### Шаги для локального запуска
1. Убедитесь, что у вас установлены Docker и Docker Compose.
2. Клонируйте репозиторий проекта:
   ```bash
   git clone https://github.com/Boraldan/Task_Management_System_simple.git
   cd <папка проекта>
   ```
3. Запустите Docker Compose:
   ```bash
   docker-compose up --build
   ```
4. После успешного запуска сервисы будут доступны по следующим адресам:
    - Keycloak: [http://localhost:8080](http://localhost:8080)
    - Gateway: [http://localhost:8765](http://localhost:8765)
    - Swagger UI: [http://localhost:8765/task/swagger-ui/index.htmll](http://localhost:8765/task/swagger-ui/index.html)
